## 스마트 홈 IoT 시스템 설계

### Part A - 시스템 구성 요소
전원 제어, 보안 감시, 환경 모니터링이 전부 가능한 시스템
#### 센서
- DHT22(온도, 습도 센서):실내 온도와 습도를 측정 -> 환경 모니터링에 사용
- PIR(모션 센서): 움직임을 감지하여 사람의 침입을 확인 -> 보안 감시에 사용
- MQ-2(가스 센서): 가스 누출을 감지하여 
*이 부분에서 센서의 이름이 생각나지 않아 AI의 도움을 받음*
#### 액추에이터
- 릴레이 모듈 (+팬, 조명): ESP32의 명령을 받아서 전원을 켜고 꺼서 전원제어를 담당
#### ESP32 장치
- ESP32 제어장치:센서 데이터를 읽어서 MQTT로 발행하고, 대시보드의 제어 명령을 구독하여 액추에이터를 작동시키는 컨트롤러
#### MQTT 브로커
- ESP32와 대시보드 사이에서 메세지를 중계, 센서 데이터 전송과 제어 명령 전달 
#### 대시보드
- Node-RED 대시보드: 센서 데이터를 게이지와 차트로 시각화, 사용자가 버튼으로 장치를 제어할 수 있게 해주는 시각적 장치

### Part B - 시스템 다이어그램
![시스템 다이어그램](https://i.imgur.com/qgJrdxG.png)
#### 데이터 흐름
- 센서 3개가 주변 환경 값을 측정
- ESP32가 센서 값을 읽고 MQTT 메세지로 변환하여 발행
- MQTT 브로커가 메세지를 받아서 중계
- Node-RED 대시보드가 구독하여 게이지와 차트 값을 표시

#### 제어 명령
- 사용자가 대시보드 스위치를 누르면 명령 ON/OFF를 발행
- MQTT 브로커가 메세지를 중계
- ESP32가 명령을 받음 -> 릴레이에 작동 신호를 보내어 팬/조명을 키거나 끈다


### Part C - 대시보드 설계
![대시보드 설계](https://i.imgur.com/sCR0vUn.png)
#### 게이지 
- 게이지 1: DHT22 센서가 측정한 실내 온도를 실시간으로 표시. 환경 모니터링용
- 게이지 2: DHT22 센서가 측정한 실내 습도를 실시간으로 표시. 쾌적도 확인
#### 차트
- 온도 추이 차트: 최근 1시간 동안의 온도 변화를 선 그래프로 표시
#### 스위치 
- 팬/조명 스위치: 사용자가 ON/OFF를 설정 가능
#### 상태표시기
- 보안 상태 표시기: PIR 모션 센서의 상태를 표시. 평소에는 초록색, 움직임이 감지되면 빨간색으로 바뀌어 침입 여부를 알림

### Part D - 자동화 시나리오 작성
#### 시나리오: 실내 온도가 30도를 넘으면 자동으로 팬을 켜고 대시보드에 경고를 표시
1. 측정: DHT22 센서가 실내 온도를 계속 측정
2. 발행: ESP32가 측정값을 읽어 sensor/temperature 토픽으로 MQTT 메세지 발행
3. 판단: Node-RED가 토픽을 구독하다가 값이 30도를 초과하는지 검사
4. 제어 명령 발행: 조건이 참이면 Node-RED가 control/relay 토픽으로 ON 명령
5. 작동: ESP32가 명령을 받아 릴레이를 닫고 릴레이가 팬을 작동
6. 대시보드 상호작용: Node-RED 대시보드의 팬 스위치가 ON으로 바뀌고 상태 표시기에 고온 경고 메세지 표시
7. 복귀: 온도가 30도 아래로 내려가면 다시 OFF 명령이 전달되어 팬이 꺼짐
*토픽 값을 어떻게 설정해야할지 모르겠어서 중간 발행 부분들은 ai의 도움을 받았습니다*

### Part E - 생각해 보기
#### 1.MQTT 브로커가 필요한 이유
> MQTT 브로커는 여러장치 사이에서 메세지를 중계하는 역할을 한다. MQTT 브로커 덕에 장치들이 서로의 IP 주소 등을 잘 몰라도 토픽만으로 통신할 수 있다. 게다가 발행/구독 방식이라서 센서 하나의 데이터를 여러 곳에서 동시에 받아볼 수 있으며 가벼운 프롵토콜이라서 ESP32 같은 장치게 적합하다.
#### 2.Node-RED가 IoT시스템에서 유용한 이유는?
> Node-RED는 노드를 드래그해서 선으로 잇는 플로우 기반 도구라서 복잡한 코드를 짜지않아도 시각적으로 데이터 처리 등을 만들 수 있다. 그리고 다양한 기능이 노드로 준비되어 있어서 빠르게 시스템 구성이 가능하다. 대시보드 노드를 사용하면 게이지, 차트, 스위치 등을 금방 생성할 수 있어서 모니터링과 제어 화면을 손쉽게 구현 가능하다.
